# 遗缺点子回收站

[English](README.md) | [中文](README.zh-CN.md)

遗缺点子回收站收集那些暂停、卡住、已经上线但增长停滞，或等待别人继续接手的软件点子。

很多项目停止，并不代表它没有价值。更常见的原因是推广困难、维护成本上升、优先级变化、没有遇到合适用户，或作者已经不再适合继续推进。

这个档案希望把项目本身和背后的经验留下来，让后来者能从真实卡点中学习，找到灵感，也发现可以继续接手的机会。

## 适合谁

- 已经做出产品，但没有找到用户或增长的独立开发者
- 暂停或放弃某个项目，希望把经验留下来的人
- 想从真实失败和卡点中学习的开发者
- 正在寻找产品灵感、开源方向或接手机会的创作者
- 关心独立开发、冷启动、推广和产品验证的人

## 这里有什么

- 被暂停或放弃的产品点子
- 已上线但增长停滞的项目
- 每个点子的简短介绍
- 作者遇到的主要卡点
- 已经完成的工作
- 技术栈、访问地址、开源地址和接手备注
- 其他开发者遇到的真实问题

这里不只收录成功故事。项目当前的状态本身也有价值，它可以帮助别人避开重复错误，重新理解一个市场，或用新的方式把一个想法继续做下去。

## 可以提交什么

你不需要等到一个项目彻底放弃才提交。以下情况都可以：

- 已上线，但没有用户或增长
- 已上线，但不再维护
- 未上线，但有原型、预览页或截图
- 未上线，但有笔记、需求说明或规划资料
- 开源项目缺少贡献者、维护者或新的负责人
- 已经完全放弃，但过程和经验值得分享

## 如何提交新的点子

1. 打开仓库的 [Issues 页面](https://github.com/ipwangxin/AbandonedIdeas/issues)。

   ```text
   https://github.com/ipwangxin/AbandonedIdeas/issues
   ```

2. 点击 [New issue](https://github.com/ipwangxin/AbandonedIdeas/issues/new/choose)。

   ```text
   https://github.com/ipwangxin/AbandonedIdeas/issues/new/choose
   ```

3. 选择 **提交遗缺点子** 表单。
4. 填写：
   - 点子名称
   - 一句话介绍
   - 访问地址，可选
   - 是否已经上线
   - 开源地址，可选
   - 卡住或放弃的原因
   - 其他原因说明，可选
   - 技术栈
   - 已经完成的工作
   - 接手备注
5. 提交后，Issue 会被标记为 `abandoned`。
6. 仓库会同步数据，并展示到首页。

## 提交建议

为了让别人更容易理解或接手你的点子：

- 不要只写“没时间”，请说明已经做到了哪一步
- 不要只写“推广失败”，请说明尝试过哪些渠道
- 如果项目已经上线，请附上产品、预览页、落地页或文档地址
- 如果项目开源，请附上 GitHub、GitLab、Gitee 或其他仓库地址
- 如果欢迎别人继续做，请说明授权范围和接手边界
- 如果这个想法不建议继续推进，请写清楚原因，帮助别人避开同样的问题

## 提交之后

提交的点子会通过公开 Issue 流程整理，并展示到网站上。描述越清楚、链接越完整、接手边界越明确，越容易被后来者理解、学习或继续推进。

手动运营时，建议优先关注前 10 个提交者：补充标签、追问关键信息、整理真实案例，并邀请因推广困难而停下来的开发者了解 Indie Traffic Alliance。
