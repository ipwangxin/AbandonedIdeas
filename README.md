# 遗缺点子回收站

这里是一个收集“被放弃的软件点子”的地方。

每个开发者手里可能都有一些没继续做下去的项目：做了一半的工具、没找到用户的产品、验证失败的想法、暂时搁置的开源计划。它们不一定没有价值，只是暂时没有遇到合适的时间、场景或接手的人。

**遗缺点子回收站** 希望把这些想法公开沉淀下来，让大家可以相互借鉴、相互启发，也让后来者有机会从别人的半成品、失败经验和真实卡点里找到新的灵感。

## 这个项目适合谁

- 有项目做了一半，但暂时不想继续维护的人
- 想复盘一次失败尝试，并把经验留给后来者的人
- 正在寻找新产品、新开源项目或副业灵感的人
- 想接盘别人遗留想法，并做出新版本的人
- 关心独立开发、冷启动、推广和产品验证的人

## 你可以在这里看到什么

- 被遗弃的产品点子
- 每个点子的核心描述
- 作者选择放弃的主要原因
- 已经完成过的工作
- 技术栈、项目链接或接盘建议
- 其他开发者遇到的真实问题

这些内容不一定都是“成功经验”，但失败原因本身也很有价值。它可以帮助后来者少走弯路，也可能让某个被放弃的想法在另一个人手里重新活过来。

## 如何提交一个新点子

1. 打开本仓库的 Issues 页面：

   ```text
   https://github.com/ipwangxin/AbandonedIdeas/issues
   ```

2. 点击 **New issue**。
3. 选择 **提交遗缺点子** 表单。
4. 按提示填写：
   - 点子名称
   - 一句话描述
   - 遗弃原因
   - 技术栈
   - 已做工作
   - 接盘建议
   - 联系方式或项目链接
5. 提交后，这个 Issue 会自动带上 `abandoned` 标签。
6. 网站会通过 GitHub Actions 同步数据，并展示到首页列表中。

## 提交建议

为了让别人更容易理解和接盘，建议你写得具体一点：

- 不只写“没时间”，也写清楚已经做到了哪一步
- 不只写“推广失败”，也写试过哪些渠道
- 如果有代码、Demo、截图或文档，可以一起附上
- 如果你愿意别人接着做，请写清楚授权和联系方式
- 如果这个点子已经不适合继续做，也可以写下原因，帮助别人避坑

## 项目机制

这个项目尽量保持简单，主要依赖 GitHub 免费能力：

- GitHub Issues：收集点子
- GitHub Issue Template：结构化提交表单
- GitHub Pages：展示网站
- GitHub Actions：同步 Issue 数据到 `docs/data.json`

没有独立后端，也没有外部数据库。先让内容流动起来，再根据真实使用情况逐步扩展。

## 维护说明

如果你是仓库维护者：

- 首页代码在 `docs/index.html`
- 联盟介绍页在 `docs/lianmeng.html`
- Issue 表单在 `.github/ISSUE_TEMPLATE/abandon_idea.yml`
- 数据同步工作流在 `.github/workflows/sync-issues.yml`
- 静态数据文件在 `docs/data.json`

手动运营时，建议优先关注前 10 个提交者：补标签、追问关键信息、整理成功案例，并邀请因推广困难而放弃的开发者了解流量互助联盟。
